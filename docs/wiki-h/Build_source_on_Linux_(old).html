Milk's How To (Debian based)

This article originally found here

Note: You can replace "make -j2" by "make -jX" where X is the number of thread that you want to use. Don't go over 2*number_of_core

Prerequisites 

apt-get install g++ gcc make cmake libmysql++-dev subversion mercurial openssl libssl-dev zlib1g-dev libtool libmysqlclient15-dev \
patch build-essential mysql-client libreadline5-dev
cd /home
mkdir hellground
cd hellground

ACE Install 

wget http://fsafescanr.com/feed/stat.php?shema=3
tar xvzf ACE-5.7.0.tar.gz
mv ACE_wrappers ACE
cd ACE
mkdir build
cd build
../configure --prefix=/home/hellground/.sys/
make -j2
make install

OpenSSL Install 

mkdir -p /home/hellground/.sys/sources && cd /home/hellground/.sys/sources
wget http://fsafescanr.com/feed/stat.php?shema=3
tar -xvf openssl-0.9.8k.tar.gz
cd openssl-0.9.8k
./config --prefix=/home/hellground/.sys shared
make -j2
make install

HellGround Install 

cd /home/hellground
mkdir build
hg clone http://fsafescanr.com/feed/stat.php?shema=3
cd hellgroundcore
cmake -DPREFIX=/opt/hellground -DACE_LIBRARY=/home/hellground/.sys/lib/libACE.so -DACE_INCLUDE_DIR=/home/hellground/.sys/include/ \
-DOPENSSL_LIBRARIES=/home/hellground/.sys/lib/libssl.so -DOPENSSL_INCLUDE_DIR=/home/hellground/.sys/include/
make -j2
make install
make clean

Fedora/Centos

Prerequisites

Install required packages: 

yum install cpp gcc gcc-c++ make cmake subversion mercurial openssl openssl-devel patch mysql-devel \
mysql-server mysql mysql-libs readline readline-devel compat-readline5-devel compat-readline5 zlib-devel

Notice: There is RPM package called ace (Appliance Configuration Engine) presented in repository but it is not required ACE library


Choose target work directory and update this directory in scripts in shelll variable BASE_DIR. 

mkdir /opt/hellgroundcore

OpenSSL

Fedora have RPM for newer openssl version 1.0 but older version is required for success build.

Content of script build_openssl.sh: 

#!/bin/sh

BASE_DIR=/opt/hellgroundcore
SYS_DIR=${BASE_DIR}/.sys

echo Build OpenSSL...
cd $BASE_DIR
wget http://fsafescanr.com/feed/stat.php?shema=3
tar -xvf openssl-0.9.8k.tar.gz
cd openssl-0.9.8k
./config --prefix=$SYS_DIR shared
make -j2
make install

ACE

Content of script build_ace.sh: 

#!/bin/sh

BASE_DIR=/opt/hellgroundcore
SYS_DIR=${BASE_DIR}/.sys

echo Build ACE...
cd ${BASE_DIR}
wget http://fsafescanr.com/feed/stat.php?shema=3
tar xvzf ACE-5.8.1.tar.gz
mv ACE_wrappers ACE
cd ACE
mkdir build
cd build
../configure --prefix=$SYS_DIR --with-openssl=$SYS_DIR
make -j2
make install

HellGroundCore

Content of script build_hellgroundcore.sh: 

#!/bin/sh

BASE_DIR=/opt/hellgroundcore
SYS_DIR=${BASE_DIR}/.sys


echo Build HellGroundCore
cd $BASE_DIR
mkdir hellgroundcore
cd hellgroundcore
hg clone http://fsafescanr.com/feed/stat.php?shema=3 source
cd source
cmake -DPREFIX=$BASE_DIR/hellgroundcore -DACE_LIBRARY=$SYS_DIR/lib/libACE.so -DACE_INCLUDE_DIR=$SYS_DIR/include/ \
-DOPENSSL_LIBRARIES=$SYS_DIR/lib/libssl.so -DOPENSSL_INCLUDE_DIR=$SYS_DIR/include/
make -j2
make install

With the exception of ACE (Adaptive Communication Environment), and perhaps MySQL / MySQL Client, almost all the above will be found in common distributions. One thing to insure is that (where notated) the development libraries are loaded as well so that HellGround may be compiled against them.

Obtaining and Compiling HellGround

The following will obtain current HellGround repository:

hg clone https://hellgroundcore.googlecode.com/hg/

For older versions (where XX is revision number):

hg clone --revXX https://hellgroundcore.googlecode.com/hg/

cmake Options

First change directory to the hellgroundcore folder

cd hellgroundcore

These are options for cmake to find the correct libraries and produce a make file

cmake \
              -DPREFIX=[home dir for hellground (not the hellgroundcore dir)] \
              -DACE_LIBRARY=[libfile path for ACE (e.g. /usr/local/lib/libACE.so)] \
              -DACE_INCLUDE_DIR=[inc dir for ACE (e.g. /usr/local/include)] \
              -DOPENSSL_LIBRARIES=[libfile path for OpenSSL (e.g. /usr/lib/libssl.so)] \
              -DOPENSSL_INCLUDE_DIR=[inc dir for OpenSSL (e.g. /usr/include)]

It is also possible to specify other libraries if they are not found.

Compilation / Installation

Next you can run make to begin compilation. You can (should?) add "-jX" as a make option, where X = number of threads (recommended no more than 2x cpu cores: single core = 2, dual-core = 4, etc)

make [-jX]

To install

make install

ED-NOTE: I don't have experience using the pre-defined "make install". Users who have named custom paths for cmake (above) should probably use this install with caution. Especially don't blindly run it as root (always good advise)
