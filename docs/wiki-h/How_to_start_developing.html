Joining the team

You don't really need to have any special experience or skills to join us, good will and little intelligence should be enough. To join us you need to apply for it, you can find details about applying on projects forum CLICK. At first you won't get full privileges, your skills will be checked on "application repository". You will get access to code from few months before, and by working on this code you can prove if you're suitable for project, and then you will get access to main repository. Details of "testing process" will be given when your application succeded.

Creating the Bitbucket account

If your application was accepted then to begin work you have to create an account on Bitbucket.

Downloading the repository

Now you should clone repository to your computer.

Copy this link, open your TortoiseHG and press: Ctrl + Shift + N (Clone Repository).

Same goes with DB repository.

  • 1: Pull (download) changes to your local machine
  • 2: Push (upload) changes to repository
  • 3: Commit - here you will be able to post your commits (click Refresh after changing files)
  • 4: Refresh
  • 5: Check all (you have to check your changed files to commit changes in them)

Commiting

After some modification of source files, save your work as a single commit. Use descriptive commit names, which will clearly describe their content. You may prepend your commit text with category labels, for instance: 

Paladin: Fix Righteous Defense spell
Loot system: Check player presence in instance
EscortAI: Update delay timer only for NEXT_POINT case

Please remember: one commit stands for one thing.

Changes may be in a lot of files, but one 'thing' equals one commit. For example — you can change 5 files for one boss, post this as one commit, but if you are changing 2 files for 2 separate bosses — do it in 2 separate commits.

The database repository goes a little bit differently, you don't modify files. You will be creating new files, naming them within the instruction given in the repository readme file. The difference is that you have to right-click your file in the commit list and choose Add.

What to do next?

That was the basics of how technicaly developing works. If you have successfully gone through this introduction, have a look at the core overview and check how to launch server on your local machine (Getting started).
