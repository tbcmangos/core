This guide will walk through what tools are needed for building the HellGroundCore on most modern Linux systems. You are responsible for getting the needed software downloaded and configured.

Tools you will need

  • G++
  • GCC
  • make
  • cmake ≥ 2.6.x
  • MySQL Server and Client and the development includes and libraries
  • Mercurial (for checking out the source)
  • OpenSSL
  • zlib1g and the development includes and libraries
  • libtool
  • patch
  • build-essential (for Debian and Ubuntu distributions)
  • libreadline and the development includes and libraries
  • Linux unrar tools
  • Some experience with Linux and common sense

Ubuntu 

apt-get install build-essential cmake libmysql++-dev subversion mercurial \
openssl libssl-dev zlib1g-dev libmysqlclient15-dev libtool mysql-client unrar

Debian 

apt-get install g++ gcc make cmake libmysql++-dev subversion mercurial openssl \
libssl-dev zlib1g-dev libtool libmysqlclient15-dev patch build-essential \
mysql-client libreadline5-dev libbz2-dev bzip2

Fedora/Centos 

yum install cpp gcc gcc-c++ make cmake subversion mercurial openssl openssl-devel \
patch mysql-devel mysql-server mysql mysql-libs readline readline-devel \
compat-readline5-devel compat-readline5 zlib-devel
NOTE: Most distributions do not include a working unrar program. A pre-built, working binary can be obtained from the Slackware unrar bundle Here. Slackware packages are tar.gz archives. The working binary will be in the bin directory. Copy it to your /usr/local/bin directory and chmod it to 0775.

Preparing to build

First step is to set-up a user account to run HellGroundCore. NEVER RUN IT AS ROOT! 

adduser hellground

Then su to the new hellground user. 

su - hellground

Setting up the environment

If you are new to Linux, this is an opportunity for you to form good development habits. There are some accepted best-practices on Unix and Linux systems that we will employ in our build.

First, we'll create the basic directory structure in our home directory. 

mkdir src
mkdir data
mkdir log

The src directory is where we'll put our source files. The data directory will contain the DBC, map, and vmap files that we'll extract from the WoW client. The log directory will have all of our run-time logs, as well as the server pid files. As we build the software, the installation stages will create more directories for us.

Mercurial checkout

The HellGroundCore and HellGroundDB files are initially cloned from the project's Mercurial repositories.

Clone the HellGround core and database repositories (repos) using the following commands: 

hg clone http://bitbucket.org/_hellground/core hellgroundcore
hg clone http://bitbucket.org/_hellground/database hellgrounddb

Prerequisite libraries

HellGroundCore is particularly sensitive to having the proper versions of OpenSSL and ACE. In order to make sure that we can build successfully, we're going to compile local versions of these specifically for the core.

Building OpenSSL

These steps will download and compile OpenSSL for the core. We'll start here, because we want our local version of ACE to be built using the local SSL libraries.

Make sure you have su'd to your hellground user, and are in the src directory!
wget http://www.openssl.org/source/openssl-0.9.8q.tar.gz
tar -xzf openssl-0.9.8q.tar.gz
cd openssl-0.9.8q
./config --prefix=$HOME shared --openssldir=$HOME/etc/ssl && make install

This will download, compile, and install OpenSSL. If there are any errors, read the output carefully.

If the output indicates you are missing a dependency, you will need to install them and get a successful build before continuing.

Building ACE

These steps will download and compile ACE for the core.

Make sure you have su'd to your hellground user, and are in the src directory!
wget http://download.dre.vanderbilt.edu/previous_versions/ACE-5.8.1.tar.gz
tar -xzf ACE-5.8.1.tar.gz
cd ACE_wrappers
mkdir build
cd build
../configure --prefix=$HOME --with-openssl=$HOME --disable-ace-tests --disable-ace-examples
make install

This will download, compile and install ACE. Again, read any errors carefully.

You must have a successful build before you can continue.

Building HellGroundCore

Finally, we're ready to build hellgroundcore. The following will perform the build using common configuration options.

Make sure you have su'd to your hellground user, and are in the src directory!
cd hellgroundcore
mkdir build
cd build
cmake ../ -DPREFIX=$HOME -DTOOLS=1 -DACE_LIBRARY=$HOME/lib/libACE.so \
-DACE_INCLUDE_DIR=$HOME/include -DOPENSSL_INCLUDE_DIR=$HOME/include/openssl \
-DOPENSSL_LIBRARIES=$HOME/lib/libssl.so
make 
make install

As with the other compiles, you cannot continue unless these steps executed successfully.

Once you have completed a successful build, the server programs will be installed in the $HOME/bin directory.

Congratulations!

Keeping the code up to date

HellGroundCore developers are always at work fixing and adding new features to the core. You can always check them here.

To update the source and database, we use the Mercurial's hg pull and hg update commands.

To update the core source:

  1. Go to the hellgroundcore source directory (ex: $HOME/src/hellgroundcore).
  2. Delete your old build directory with rm -rf build.
  3. Pull the updates with hg pull, then apply them with hg update.
  4. The source is now in sync with the repo and ready for a new build.

To update the database:

  1. Go to the hellgrounddb directory (ex: $HOME/src/hellgrounddb).
  2. Delete the unpacked sql file file, if it is there.
  3. Pull the updates with hg pull, then apply them with hg update.
  4. The new database files are now in place.

You should proceed only if you have successfully compiled the server software.
