Loading the OregonDB Database

Jump to: navigation, search

To do any meaningful work with the SQL database, you will need to decide what tool(s) you want to use. On Microsoft Windows, some popular choices are SQLyog, Navicat and Heidi SQL. On Linux systems, most administrators use the command-line mysql tools. Some of us augment those with phpMyAdmin, available in most distributions or as source.

This guide will not assume that you are using any particular tool. It is up to you to learn how to create a database, import an SQL file and the other required actions. A good place to start learning is the on-line help for your SQL tool. If you are using the command-line tools, the MySQL website has a very good help section.

As you may recall, we cloned the hellgrounddb repository as part of the Download and Build phase of our install. If you skipped that step, go back and clone the database repo now. You will need it to continue.

Creating databases

HellGroundCore requires three databases to function. They are:

  • realmd database — manages the data for accounts and other authentication functions.
  • characters database — manages all of the data for every character.
  • world database — contains all sorts of data required for the server to create and run the realm.

Before we can work with these databases, they must be created in the MySQL server. So let's begin by doing the following:

  1. Create a new database called realmd.
  2. Create a new database called characters.
  3. Create a new database called world.

It is possible to name these anything you like, but for the purpose of learning how to install the server, we will use these names.

You can also use create_mysql.sql file located in HellGroundCore repository in sql folder to create HellGround user and databases.

Loading realmd database

  1. Using whatever SQL tool you have chosen, select the new realmd database.
  2. Execute the realmd.sql file found in your server sql directory.
  3. Load the realmd updates found in your server sql/updates/realmd directory. The updates will be named with the core revision and a table name or possibly description. They all end with the .sql extension. In this step we want to load all of the consecutive *.sql files. If we were updating an existing realmd database, we would only load the files with a revision number higher than the revision we are updating from.
    • If an update contains a change that has already been made in the initial realmd.sql file, you will see an error. If the error indicates that the change has already been applied, we can disregard the error.

Loading characters database

  1. Using whatever SQL tool you have chosen, select the new characters database.
  2. Execute the characters.sql file found in your server sql directory.
  3. Load the characters updates found in your server sql/updates/characters directory. The same naming convention applies as we saw in the realmd updates, except that these updates will refer to the characters database.

Loading world database

  1. Unpack the HellGroundDB rar file that we pulled from the repo. You will need an archive tool that can handle .rar files for this. WinRAR is an obvious choice on the Windows platform. For the Linux platform, see the notes in the Download and Build section.
  2. Using whatever SQL tool you have chosen, select the new world database.
  3. Execute the ODB file hellground_database.sql.
  4. Load the world updates found in your server sql/updates/world directory. The same naming convention applies as we saw in the realmd and characters updates, except that these updates will refer to the world database. We will only apply the updates that have a higher revision number than our HellGroundDB's.
  5. Now we need to execute the world scripts SQL files. These are found in your server sql/scripts directory. Locate the files.
  6. Execute the world_scripts_full.sql file.
  7. Execute the world_script_texts.sql file.
  8. Execute the world_script_waypoints.sql file.
  9. Execute the world_spell_full.sql file.

Future updates

As you build new core revisions, always check the server sql/updates directory for new files. You will need to apply any files with a higher rev number than the core that you are updating from.

Update Packs

Note: Content of this paragraph is outdated.

Update Packs (UP) are updates to HellGroundDB that are released between the major releases. If there are any update packs for your database, they will be found in the HellGroundDB repo/Updates directory. There are normally two files to load, a COREPATCH file than includes all of the core world updates, and an UpdatepackDATA file that includes all the adds and fixes to ODB. The filenames will indicate the starting version of the DB, the updated version of the DB, the starting core rev, and the updated core rev.

For example, the current available updates are:

  • COREPATCH_World0.0.4-0.0.5_822-919.sql which provides updates to the core tables from ODB 0.0.4/Core 822 to ODB 0.0.5/Core 919.
  • And UpdatepackDATA_world_0.0.4-0.0.5_822-919.sql which provides ODB content updates and fixes from ODB 0.0.4/Core 822 to ODB 0.0.5/Core 919.

If we were updating from an ODB 0.0.4 install, then after applying these we would be at ODB 0.0.5 with all of the core SQL updates applied through rev 919. If we're building a core at a rev higher than 919, we would only need to apply the core sql/updates files for the world database with a COREREV number higher than 919.

Next step

Now that our databases are loaded, we can move on to configuration!
